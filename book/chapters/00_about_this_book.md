# About this book

This book is my learning log and playbook for becoming an AI engineer from scratch. It is intentionally practical: I learn concepts, implement them, and write down what worked (and what didn’t) in a way I can reuse later.

## What this book will cover

The content is organized as multiple learning tracks that you can follow in parallel:

### LLM learning

- How LLMs work at a high level (tokenization, attention, training vs inference).
- Prompting patterns and evaluation.
- RAG fundamentals (chunking, embeddings, retrieval, reranking).
- Fine-tuning basics and when not to fine-tune.

### AI agents

- Tool use, function calling, and structured outputs.
- Planning, memory, and retrieval for agents.
- Building reliable workflows (retry, guardrails, observability).
- Shipping small agent projects end-to-end.

### LeetCode (interview-style problem solving)

- Patterns: two pointers, sliding window, stack/monotonic stack, BFS/DFS, DP.
- How to practice deliberately (review loops, mistake journals, spaced repetition).
- Turning problem patterns into reusable templates.

### Data structures & foundations

- Big-O thinking, tradeoffs, and common pitfalls.
- Arrays/strings, hash maps/sets, heaps, trees, graphs.
- Writing clean, testable Python.

### Machine learning learning

- Core ML concepts: bias/variance, metrics, cross-validation.
- Classic models: linear/logistic regression, trees, boosting.
- Feature engineering and leakage.
- A minimal MLOps mindset: reproducibility, data/versioning, evaluation.

### Cloud engineering learning

- Packaging and running ML/AI apps.
- Basic deployment patterns (containerization, simple hosting, CI/CD).
- Practical considerations: secrets, configuration, monitoring.

## Goals

- Build strong fundamentals that transfer across tools and frameworks.
- Learn by building small projects and reflecting on outcomes.
- Keep reproducible notes and code that can later be published as a book.

## Who this is for

- You are learning AI engineering and want a structured, realistic path.
- You want a notebooks-first workflow that blends explanation + code.
- You prefer practical examples over theory-only explanations.

## How to use this book

- Read the chapter objectives first.
- Run the notebook cells and modify them.
- Do the exercises; they are designed to create long-term skill.
- Keep your own “mistake journal” notes in the notebooks.

If you already have experience, you can skim foundations and jump to the tracks you care about.

## Conventions

- Each chapter starts with objectives.
- Each chapter ends with exercises and references.
- Code should be runnable and reproducible.
- When something breaks, the fix and root cause should be recorded.
